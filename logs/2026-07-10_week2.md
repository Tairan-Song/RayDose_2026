## First Functional Pipeline

The first local sanity run verified that the pipeline can train, evaluate, and
export prediction files. The run was subset-based and is intentionally not
published as benchmark evidence.

## Interpretation

The pipeline is functional, but this local run is a subset sanity check rather
than a full-dataset benchmark. It should not be reported as final baseline
performance.

The current model is useful as a technical baseline:

- the data pipeline works
- training runs on GPU
- checkpoints are created
- validation predictions are exported
- `Dose_Bx_CPy.mha` files are generated
- dose-specific metrics are produced

The full-dataset baseline has not been completed yet.

## Next Steps

- Run a full-dataset or clearly scoped medium-scale baseline before publishing
  performance claims.
- Increase the dose-support loss weight, for example `mask_weight = 5` or `10`.
- Add an explicit beam geometry mask/channel.
- Compare HU input against HU-to-density input.
- Compare no-energy conditioning against energy-spectrum conditioning.
- Try sliding full-volume inference.
- Train longer and increase training sample coverage after improving spatial conditioning.
